    My role

  • 0→1 design system for 3D lightfield displays
  • Complete component library with depth-aware patterns
  • Brand system refresh (logo, wordmarks, color architecture)
  • Technical documentation and implementation guidelines
  • Cross-functional adoption across 50+ designers and engineers

When Leia launched their 3D lightfield display technology in 2021, no existing design patterns addressed how interfaces should behave in three-dimensional space. Depth became a new design dimension alongside color, typography, and layout. I established their foundational design system from scratch, defining how 50+ designers and engineers would build for this entirely new medium.

The system introduced standardized depth zones, 3D-aware components, and motion principles that would shape 10+ apps and services across Leia's product ecosystem. This wasn't about adapting existing patterns—it required building scalable infrastructure for a display technology that had never existed before.

I started by establishing the foundational patterns that would govern the entire system. Working within an 8pt grid informed by the tablet's resolution constraints, I defined 3D depth zones—a systematic approach to layering UI elements in three-dimensional space. Each component needed precise specifications for depth placement, contrast ratios, and scale to ensure readability in lightfield viewing conditions.

The color system extended Material Design's semantic variables with depth-specific adjustments. Typography established baseline grid alignment that worked across all depth planes. This technical foundation ensured every designer could build interfaces that felt cohesive, regardless of which app they were working on.

Building the component library meant solving unprecedented technical challenges. I used Material Design as the foundation—since Leia's Android platform required Material conventions—but every component needed depth-specific adaptations. Buttons required higher contrast ratios than Material guidelines recommended because depth perception fundamentally changes how users perceive color relationships.

I built navigation systems optimized for landscape tablet use, modular media controls with appropriate scale for 3D viewing, and extended Material's icon library with consistent construction rules. Each component included implementation guidelines that documented not just visual specs, but the technical reasoning behind depth placement and contrast decisions. This ensured engineers understood why components behaved differently in 3D space.

The brand system refresh ran parallel to the design system work, ensuring visual consistency across both product interfaces and marketing materials. I refined logo proportions for better optical balance, created custom product wordmarks with consistent stroke weights and proportions, and consolidated the color palette using semantic variables that connected to Material Design's layering system.

This wasn't cosmetic work—it was systems architecture. The brand tokens I defined became the foundation for how every designer applied color, typography, and visual hierarchy across Leia's entire product ecosystem.

I redesigned the app icon family to ensure each icon clearly communicated its function while maintaining visual unity across the ecosystem. The LeiaChat icon, for example, uses a camera's profile angle to represent "video with depth"—a visual metaphor specific to 3D communication. Each icon followed consistent construction principles that reinforced Leia's brand identity while remaining functionally distinct.

Building the system was only half the challenge—driving adoption required a systematic rollout strategy. I implemented the system app-by-app using a phased approach, conducting 1:1 working sessions with designers to ensure they understood not just what to build, but why depth and contrast behaved differently in 3D space.

I collaborated with engineering teams to establish design-code parity, ensuring our Figma components mapped directly to Android implementations. Weekly stakeholder presentations maintained alignment across product, engineering, and leadership. Within six months, the system achieved cross-functional adoption, with design-to-development time reduced by an estimated 75% as teams no longer rebuilt components from scratch.
